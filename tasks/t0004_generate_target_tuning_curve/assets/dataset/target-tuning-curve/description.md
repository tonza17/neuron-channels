# Target Direction Tuning Curve (synthetic)

## Metadata

* **Name**: Target Direction Tuning Curve (synthetic)
* **Version**: 1.0.0
* **Year**: 2026
* **Authors**: Anton Nikolaev 🇬🇧 (University of Sheffield)
* **License**: CC0-1.0
* **Access**: public
* **Size**: 12 angles × 20 trials (12 mean rates, 240 per-trial rates)
* **Preferred direction θ_pref**: 90°
* **Baseline rate r_base**: 2.0 Hz
* **Peak rate r_peak**: 32.0 Hz
* **Sharpness exponent n**: 2.0
* **Noise σ**: 3.0 Hz (Gaussian, clipped at 0)
* **Random seed**: 42
* **Closed-form DSI**: 0.8824

## Overview

This dataset is the canonical target direction-selectivity tuning curve that every downstream
tuning-curve fitting task in this project will optimise against. Rather than digitising a curve from
a published figure — which would introduce digitisation error and licensing friction — the
researcher chose to synthesise a clean, parameterised curve from a closed-form analytical
expression. This makes the ground-truth behaviour exactly known, gives the `tuning_curve_rmse`
metric a stable reference across every experiment, and lets `tuning_curve_reliability` compute a
meaningful noise baseline because the per-trial noise distribution is fixed.

The curve is a half-wave-rectified cosine raised to the power `n = 2`, with preferred direction at
90°, baseline 2 Hz, and peak 32 Hz, producing a closed-form direction-selectivity index (DSI) of
0.882 — squarely within the 0.6-0.9 biologically plausible band requested by the task. Twenty
independent Gaussian-noise trials per angle (σ = 3 Hz, clipped at 0 Hz) are generated with a fixed
seed so the dataset is exactly reproducible from the parameters alone.

## Content & Annotation

The dataset contains three files that together fully describe the target curve:

* `files/curve_mean.csv` — one row per sampled direction, with columns `angle_deg` (0 to 330° in
  30° steps) and `mean_rate_hz` (the closed-form mean rate at that angle, not a noisy empirical
  mean). This is the deterministic target that `tuning_curve_rmse` compares fitted simulations
  against.
* `files/curve_trials.csv` — one row per (angle, trial) pair. Columns are `angle_deg`,
  `trial_index` (0 to 19), and `rate_hz` (the noisy observation for that trial). All 240 rows
  together constitute the synthetic "observed data" that a downstream fitting task would see.
* `files/generator_params.json` — the exact dataclass fields used to generate the curve, so the
  data can be regenerated byte-for-byte from `code/generate_target.py` + these parameters.

No manual annotation is involved — the dataset is fully synthetic. The "annotation" is the
closed-form formula, which is explicitly stored in `details.json.generator_params.functional_form`.

## Usage Notes

* This dataset is a **target**, not training data. Downstream tasks should compare the firing-rate
  curve produced by their model against `curve_mean.csv` using `tuning_curve_rmse`, and against
  `curve_trials.csv` using `tuning_curve_reliability`.
* Load `curve_mean.csv` and `curve_trials.csv` with `pandas.read_csv` using the dtype
  `{"angle_deg": float, "trial_index": int, "rate_hz": float, "mean_rate_hz": float}` to avoid
  pandas' dtype-inference surprises.
* The 0° row is the null direction (θ_pref = 90°, so null = 270° in the data; 0° is 90° away
  from the peak). Do not reorder the rows to "start at the preferred direction" — downstream tasks
  assume rows are in ascending `angle_deg`.
* Reproducing the curve exactly requires `numpy.random.default_rng(seed=42)` on numpy ≥ 1.17. The
  seed and everything needed to regenerate the data are stored in `generator_params.json`.
* The curve sits at the upper end of the biologically plausible DSI range. If a later task needs a
  weaker-DSI reference, do **not** edit this asset — create a new dataset asset (e.g.,
  `target-tuning-curve-weak-dsi`) with different parameters and document the relationship through a
  corrections file if needed.

## Main Ideas

* The dataset is purely synthetic; its ground-truth formula and parameters are stored explicitly
  with the data, so reproducibility is perfect.
* The per-trial table lets `tuning_curve_reliability` compute a realistic noise floor without
  relying on a second physical dataset.
* DSI = 0.882 is near the top of the biologically plausible band — weaker-selectivity variants
  should be added as separate assets rather than by editing this one.
* Angle rows are in ascending `angle_deg` order starting at 0° (not at the preferred direction) and
  must not be reordered.

## Summary

`target-tuning-curve` is the canonical, synthetic direction-tuning reference that every later
optimisation task in this project will compare its simulated responses against. It consists of a
closed-form cosine-raised-to-power curve (r_base = 2 Hz, r_peak = 32 Hz, θ_pref = 90°, n = 2)
sampled at 12 evenly-spaced directions, plus 20 Gaussian-noise trial replicates per direction (σ =
3 Hz, seed = 42). All parameters are stored alongside the data so the dataset is exactly
reproducible.

In the project, this asset fixes the target behaviour for the `tuning_curve_rmse` metric (how far a
simulated curve deviates from the target mean) and gives `tuning_curve_reliability` a concrete noise
floor (since the per-trial rates are the synthetic "observations" the metric expects). The DSI of
0.882 places it at the top of the 0.6-0.9 biologically plausible band, so any fit that recovers this
DSI is clearly matching a direction-selective response.

The alternatives would have been to digitise a published tuning curve (rejected because of
digitisation error and opaque noise statistics) or to use a von Mises parameterisation (rejected
because the shape parameter κ is harder to interpret than the raised-cosine exponent n). If later
work needs additional reference curves with different DSI levels, preferred directions, or
sharpnesses, they should be added as separate dataset assets rather than modifying this one.
